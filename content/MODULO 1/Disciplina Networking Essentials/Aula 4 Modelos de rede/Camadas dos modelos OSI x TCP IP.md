# Camadas dos modelos OSI x TCP/IP

Tanto o modelo OSI quanto o modelo TCP/IP possuem algumas semelhanças. Ambos se baseiam em camadas de protocolos que funcionam de maneira independente. É nesse funcionamento em camadas que encontramos as usas semelhanças. 

Nos dos casos, existem dependência entre as camadas próximas!

É a camada anterior que transmite as informações decodificadas à camada seguinte, assim sucessivamente. Muito embora eles possuam essas semelhanças, são em si modelos cujo funcionamento opera de forma muito diferente. Pode-se pensar numa comparação por meio dos modelos de referência. 

Modelo OSI tem seu uso como referência, padrão, para a comunicação de rede. Nesse modelo, há o funcionamento de sete camadas que se intercomunicam umas com as outras. Pode-se ter como conceito central e principal do funcionamento desse modelo três funções:
	Serviços
	Interfaces
	Protocolos 

É função do modelo OSI fazer distinção entre cada uma dessas funções. 

No modelo OSI, existem sete camadas com funções distintas. O modelo OSI distribui suas camadas conferindo a cada uma delas uma função que será exercida de acordo com os dados recebidos. Uma camada depende da outra para o pleno funcionamento. 

A transmissão de rede é desmembrada nas 7/4 camadas do emissor, e reconstruída nas 7 do receptor.

Esse modelo também evita que as modificações feitas em uma camada afetem a outra, o que possibilita maior rapidez e agilidade no seu desenvolvimento. No modelo OSI, são as camadas inferiores que fornecem serviços para as camadas superiores. Além disso, cada camada é responsável por uma atividade, e vai passando em sequência para as demais camadas.

No modelo OSI não existem restrição ao uso dos protocolos, mas no TCP/IP cada camada tem sua lista de protocolos comuns!

Uma vez que cada camada possui um funcionamento específico, entende-se que cada protocolo designado a cada uma das camadas é de inteira responsabilidade dela também quando se levam em consideração os pares de protocolos.

`Enquanto na camada de transporte OSI,` o serviço é transparente e visível aos usuários e possuí somente COC, o modelo TCP/IP suporta ambos os tipos de transporte. Sendo assim, o modelo TCP/IP deixa a critérios do usuário escolher. `De Forma diferente à do modelo OSI, o modelo TCP/IP foi desenvolvido com quatro camadas.` 

Algumas das camadas dos modelo TCP/IP possuem o mesmo nome das camadas do modelo OSI, em algumas situações elas se equiparam quanto ao seu funcionamento.
